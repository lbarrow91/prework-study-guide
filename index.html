HTML

  • The head element contains information about the webpage
  • The body element represents the visable content shown to the user
  • Section elements are semantic elements that define a section in a document that contains thematic content
  • An element refers to the entire element from opening tag to closing tag
  • A tag refers only to what apppears inside the angle brackets
  • The element that holds all the other elements is known as the parent element - such as the BODY element
  • Elements that are nested inside other elements are called child elements

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

  • When a developer wants to style an element directly in the HTML file, they use inline CSS
  • Inline CSS is useful for quick and permanent changes but is less flexible because each inline style must be edited individually whenever you make a design change
  • Internal CSS style sheet can be applyed to a whole parent element such as 'Head' - the rules only apply to that page and a single change to the CSS rule will apply to all tagged page elements
  • An external CSS style sheet is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier
  • We use a 'link' element to link the external CCS style sheet within the HTML
  • Declarations contain two important components: the CSS property we want to apply and the value of the property

  • The padding property adds space around the content inside an element inside the border

Git

  • git status: checks what branch we are currently on
  • git checkout -b brand-name: creates a new branch and switches to it
  • git branch: lists your branches
  • git checkout: Switches to another branch
  • git add -A: Adds changed files in the working branch (stored locally) to the staging area of Git
  • git commit -m "comment on what has been done": Commits changes which have been staged to the repository
